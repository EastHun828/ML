1. 일반적인 파라미터
① criterion
•	설명: 분할 기준을 결정하는 함수
•	기본값:
o	gini (Classification)
o	squared_error (Regression)
•	옵션:
o	gini: 지니 계수 (클래스 불순도를 최소화)
o	entropy: 엔트로피 (정보 이득을 최대화)
o	log_loss: 로지스틱 손실 (확률 기반 의사결정)
o	squared_error: 평균제곱오차 (MSE, 회귀)
o	friedman_mse: 프리드먼 MSE (회귀)
o	poisson: 포아송 분포 (회귀)
② splitter
•	설명: 노드를 분할하는 전략
•	기본값: best
•	옵션:
o	best: 최적의 분할을 찾음
o	random: 무작위 분할

2. 트리 깊이 관련 파라미터
③ max_depth
•	설명: 트리의 최대 깊이 지정 (과적합 방지)
•	기본값: None (모든 리프 노드가 순수해질 때까지 성장)
•	권장 설정: 너무 깊으면 과적합, 너무 얕으면 과소적합 가능 → GridSearchCV로 최적화 추천
④ min_samples_split
•	설명: 노드를 분할하기 위한 최소 샘플 수
•	기본값: 2
•	권장 설정:
o	값이 크면 과소적합 위험
o	기본값은 2이지만, 과적합 방지를 위해 5~10 정도 추천
⑤ min_samples_leaf
•	설명: 리프 노드에 있어야 하는 최소 샘플 수
•	기본값: 1
•	권장 설정:
o	작은 값 → 모델이 복잡해질 가능성 (과적합 위험)
o	큰 값 → 모델이 단순해짐 (과소적합 가능)
o	일반적으로 1~5 정도 추천
⑥ max_leaf_nodes
•	설명: 최대 리프 노드 개수 제한
•	기본값: None (제한 없음)
•	권장 설정:
o	너무 크면 과적합
o	너무 작으면 과소적합 가능
________________________________________
3. 특성 선택 관련 파라미터
⑦ max_features
•	설명: 노드를 분할할 때 고려할 최대 특성 개수
•	기본값: None (모든 특성을 사용)
•	옵션:
o	None: 모든 특성 사용
o	auto or sqrt: 전체 특성 개수의 제곱근 (Classification 기본값)
o	log2: 전체 특성 개수의 로그 값
o	int: 특정 개수의 특성만 사용
o	float: 전체 특성 개수의 비율만큼 사용
________________________________________
4. 트리 성능 향상 관련 파라미터
⑧ ccp_alpha (Cost Complexity Pruning)
•	설명: 가지 치기를 통해 모델의 복잡도를 조절하는 파라미터
•	기본값: 0.0 (가지 치기 없음)
•	설정 추천: GridSearchCV로 최적값 탐색 필요
⑨ random_state
•	설명: 랜덤 시드 설정 (재현 가능성 보장)
•	기본값: None
•	추천값: 특정 값을 설정하면 같은 결과를 얻을 수 있음 (예: 42)
